# Release Plan & Roadmap

## 1. Delivery Model

- Iterative "vibe-coded" development with clear quality gates.
- Two-stage planning: MVP + Expansion.

## 2. MVP Milestones

### M1 - Foundations

- Repo setup, docs baseline, CI pipelines.
- Core architecture skeleton.
- Localization framework setup.

### M2 - Core Gameplay

- Board rendering and swap mechanics.
- Across/down clue interactions.
- Win/lose states and score.

### M3 - Content & Quality

- Algorithmic generator with quality gates.
- Curated multilingual baseline dataset.
- Family/kid filtering profile support.

### M4 - Polish & Release Readiness

- Accessibility hardening (WCAG 2.1 AA target).
- Performance optimization for mobile.
- Cloudflare deployment guide and production checks.

## 3. Phase 2 (Post-MVP)

- Account system (Google/Apple/Email).
- Cloud saves and cross-device continuity.
- Global leaderboards and enhanced anti-cheat.
- Multiplayer versus mode.
- Internal authoring tools.

## 4. Phase 3 (Long-Term)

- Live events, challenges, seasonal content.
- Advanced personalization and social loops.
- Extended theme packs and sound/music system.

## 5. Release Criteria

A release is approved when:

1. Required CI/CD checks pass.
2. Security and accessibility checks pass.
3. Documentation and changelog are updated.
4. No unresolved high-priority defects remain.
