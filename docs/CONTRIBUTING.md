# Contributing Guide

Thank you for helping improve SwapPuzzle.

## 1. Ground Rules

- Be respectful and constructive.
- Keep pull requests focused and small when possible.
- Update docs whenever behavior or architecture changes.

## 2. Branching Model

- `main`: stable branch.
- Feature branches: `feature/<short-name>`.
- Fix branches: `fix/<short-name>`.

## 3. Commit Style

Use conventional commits:

- `feat:` new feature
- `fix:` bug fix
- `docs:` documentation only
- `test:` test changes
- `refactor:` code improvements without behavior change
- `chore:` tooling/maintenance

## 4. Pull Request Checklist

- [ ] Linked task from `docs/TASK_TRACKER.md`
- [ ] Tests added/updated
- [ ] Accessibility impact reviewed
- [ ] Security impact reviewed
- [ ] Docs updated
- [ ] Changelog entry added

## 5. Definition of Done (Contribution Level)

A contribution is done when:

1. CI checks pass.
2. Reviewer approvals are complete.
3. Documentation and changelog are updated.
4. No unresolved high-severity security warnings.

## 6. Beginner-Friendly Code Requirement

Please write explanatory comments so less experienced developers can understand intent and data flow quickly.
