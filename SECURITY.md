# Security Policy

## Supported versions
The `main` branch is supported with security fixes.

## Reporting a vulnerability
Please do **not** open a public issue for vulnerabilities.

- Email: **security@swappuzzle.example**
- Include: reproduction steps, impact, affected routes/files, and suggested mitigation if available.
- Response target: acknowledgment within 72 hours.

## Disclosure process
1. Triage and severity classification.
2. Fix and verification.
3. Coordinated disclosure and changelog note.
